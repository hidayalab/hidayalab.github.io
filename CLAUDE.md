# CLAUDE.md

## Project Overview
HidayaLab is a digital Islamic technology company.  
We build software products that serve the Muslim community, such as:
- Masjid management systems
- Quran memorization apps
- Productivity tools (e.g., HidayaDesk — bookmarks, solat reminders, Quran/hadith widgets)

Our mission is to combine modern technology with timeless Islamic values to empower individuals and institutions.

---

## Purpose of this Document
This file gives Claude (and other AI assistants) guidance on how to contribute to HidayaLab projects in a way that aligns with our goals, values, and coding standards.

---

## Contribution Guidelines

1. **Tone & Language**
   - Maintain a professional, respectful, and faith-conscious tone.
   - When mentioning Islamic terms, spell them fully and respectfully (e.g., "Assalamu Alaikum wa Rahmatullahi wa Barakatuh").
   - Avoid slang, sarcasm, or culturally inappropriate references.

2. **Code Standards**
   - Default to clean, modular, and documented code.
   - Prefer readability over cleverness.
   - Use lowercase for repo names (`faithdesk`) but capitalize product branding (`FaithDesk`) in docs.

3. **UI/UX Design**
   - Favor simple, modern, and minimal designs.
   - Use accessible color contrasts.
   - Where appropriate, incorporate subtle Islamic-inspired aesthetics (geometric patterns, calligraphy motifs, calming tones).

4. **Content & Examples**
   - If providing sample data (e.g., Quran verses, Hadith), use authentic references and mark them clearly.
   - Avoid fabrications or unverified religious content.

5. **File Naming**
   - Use kebab-case for files (e.g., `masjid-dashboard.js`).
   - Use PascalCase for React components (e.g., `PrayerWidget.jsx`).

---

## Development Principles
- **Faith-first**: Products should respect Islamic values and ethics.
- **User-focused**: Build tools that are simple, helpful, and intuitive.
- **Scalable**: Design systems that can grow with the community’s needs.
